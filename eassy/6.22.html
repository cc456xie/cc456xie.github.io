# 第一次作业

# 1，谈谈我是如何理解JS异步编程的，EventLoop、消息队列都是做什么的，什么是宏任务，什么是微任务？

# 1.1，我是如何理解JS异步编程的

因为浏览器的特性，JS被设计为一门单线程的语言。也就是说整个执行流都在一个线程上，这样设计有利于利用JS开发网页应用，但是也会带来一些问题。比如假如JS脚本以同步模式编译执行，那么如果在其中有一些操作比较耗时，那么整个执行流都会被这个耗时操作阻塞，导致整个脚本迟迟无法执行完，用户的体验也会大打折扣。 所以，异步编程就是为了解决这个问题，所谓异步编程，就是将JS中的耗时操作，每次遇到一个耗时操作后会先将其执行，不等待他执行完毕，便继续执行后续的操作，因为浏览器是多线程的，可以利用浏览器在后台执行这个操作，同时他执行完后的回调函数放入消息队列，等待当前执行栈空闲，才会从消息队列中去除这个回调函数。 这样一来，耗时操作便不会阻塞程序的运 行。 目前的异步编程方式一般有回调函数,promise，Generator，async await等

# 1.2 event loop 和 消息队列

event loop即事件循环，事件循环和消息队列是用来维护页面中的大部分任务（即事件）的一种机制。 JS引擎在遇到需要异步执行的事件（DOM事件监听，网络请求，setTimeout等...）后，会将其放入对应的异步调用线程中执行(在浏览器中为Web APIs)，在执行完毕后会在消息队列中注册一个回调函数，等待被执行。当完成这一次事件循环后，就会从消息队列中取出回调函数压入执行栈，开启下一次事件循环。

# 1.3 什么是宏任务和微任务

宏任务和微任务都是异步任务。为了更好地区分异步任务的执行顺序，便分化出了宏任务与微任务。

  • 宏任务：包括主代码块，setTimeout，setInterval，setImmediate等

  • 微任务：promise，process.nextTick等 所有的微任务都是在宏任务执行期间被压入执行栈的，在宏任务执行期间的所有微任务，都会在下一个宏任务从消息队列取出前被执行。

//support.js
class Container{
	static of(value){
		return new Container(value)
	}
	constructor(value){
		this._value = value
	}
	map(fn){
		return Container.of(fn(this._value))
	}
}
    
class  Maybe {
	static of(x){
		return new Maybe(x)
	}
	isNothing(){
		return this._value === null || this._value === undefined
	}
	constructor(x) {
		this._value = x
	}
	map(fn){
		return this.isNothing()? this:Maybe.of(fn(this._value))
	}
}

module.exports = { Maybe,Container}
