# typescript

# 强类型与弱类型

非权威定义：无法进行隐式类型转换的语言为强类型，否则为弱类型

# 动态类型与静态类型

定义：可以随时修改变量类型的语言为动态类型,变量类型在声明时就定义好切无法随意转换即为静态类型

# js的类型系统

JavaScript是动态类型,弱类型语言

早期JavaScript规模较小，没有必要设计为强类型，静态类型语言。后来随着js愈发流行，前端开发领域高速发展，弱类型/静态类型已经不能很好的满足大型的复杂的项目开发。

# JavaScript 弱类型的问题

  • js是脚本语言，没有编译环节，也无法进行类型检查，所以只有在运行阶段才会发现类型错误，而不是在编译阶段就检查出错误

  • 隐式类型转换可能带来意想不到的结果：

function sum (a,b)
{
	return a + b
}

console.log(sum(10,10))  //20
console.log(sum(10,'10'))   //1010
  • 对象索引错误用法：
const obj = {}
obj[true] = 100

console.log(obj['true'])  //100,  这里通过字符串的'true'索引到了布尔值的true，这在项目开发中是不利的用法

# 强类型的优势

  • 错误更早暴露

  • 明确的类型可以让代码智能提示更准确

  • 重构更可靠，比如修改变量名称后，强类型可以立即报出错误，可以迅速定位到需要修改的地方

# typescript概述

  • typescript是JavaScript的超集

# typescript常见配置

tsconfig.json

  • target : ES5 使用什么JS规范来编译js文件

  • module : commonjs 使用什么模块化规范

  • outDir : dist 编译结果输出的文件夹

  • rootDir : src 源代码（ts文件）存放的目录

  • sourceMap : true 开启源代码映射

  • strict : true 严格模式

# 标准库

定义：内置对象所引用的文件，若未引入对应的标准库，那么使用其内置对象时会报错，如ES2015,DOM等

# 作用域问题

定义在不同文件的同名变量会处在同一作用域（全局作用域），所以无法二次声明

// a.ts
const a = 100
// b.ts
const a = 100 //err

解决方法：

  • 用立即执行函数包裹住变量，创造一个独立的函数作用域

  • 导出该文件，使其变成一个独立的模块化作用域

# Object类型

typescript中有一种 object类型，该类型不仅可以接受 对象的值，还可以接受函数、数组。

const foo:object = {} | [] | function () {}

若要声明一个只能赋值为对象的变量，可以使用如下方式：

const foo:{ bar:number, baz:string} = { bar:100, baz:'typscript' }  //变量的值必须与声明完全一致

# 数组类型

//声明一个只含数字的数组，有以下两种方式

const arr1:Array<number> = [1,2,3]

const arr2:number[] = [1,2,3]

# 元组（Tuple）

元组是一种特殊的数组，是明确了元素数量，以及每个元素的类型的数组

const tuple: [number,string] = [100,100] //err

const tuple: [number,string] = [100,100,'xhj'] //err

const tuple: [number,string] = [100,'xhj']

# 枚举类型（Enum）

//声明一个枚举

//若未赋值，则会自动从零自增,或者从第一个赋值元素开始累加
enum status {
	status1,
	status2,
	status3
}

//赋值则按照赋值为准
enum status {
	status1 = 0,
	status2 = 1,
	status3 = 2
}

//字符串枚举
enum status {
	status1 = 'aaa',
	status2 = 'bbb',
	status3 = 'ccc'
}

  • 枚举类型会入侵js代码：枚举类型会被tsc编译为一个键值对对象，可以通过索引器的方式访问枚举 如: status[0] //status1

  • 如果不想使用索引器寻找枚举的键，可以在枚举类型声明前加 const，成为常量枚举类型:

const enum status {
	...
}

此时编译后的枚举类型不会是键值对对象，而是普通的数值

# 函数类型

  • 函数声明：
//接受两个数字，返回一个字符串
function fun1(a:number,b:number): string {
	return 'string'
}

//可选参数 ？ ，可选参数必须出现在参数列表的最后
function fun1(a:number,b?:number): string {
	return 'string'
}

//参数默认值 ，默认参数必须出现在参数列表的最后
function fun1(a:number,b:number = 10): string {
	return 'string'
}

/接受任意数量的参数，可使用ES6的rest操作符
function fun1(a:number,b:number,...rest:number[]): string {
	return 'string'
}
  • 函数表达式：
const fun2 = function(a:number,b:number):string {
	return 'string'
}

//或者
const fun2:(a:number,b:number) => string

# 任意类型

ts使用any接受任意类型的值

function stringify(value: any){
	return JSON.stringify(value)
}
//由于JSON.stringify接受任意类型的参数，所以可以使用any来定义参数类型

# 类型断言

明确某个成员的类型，类型断言 ≠ 类型转换，类型转换在运行阶段，类型断言在编译阶段

const arr = [10,20,30,40]

const res = arr.find(i => i > 0)

const num1 = res as number //第一种方式 as， 将res断言为 number类型，推荐使用

const num2 = <number>res   //第二种方式，<>可能有兼容问题。JSX下不能使用

# 接口（interface）

定义： 一种对数据结构的规范

  • 实现一个接口：
interface Post {
	title:string
	content:string
}

function post(post:Post)
{
	console.log(post.title)
	console.log(post.content)
}

post({
	title:'hello typescript',
	content:'ccc'
})

上面的接口显式要求了post函数需要传入title和content作为参数，接口编译之后不会存在

  • 接口的可选成员：
interface Post {
	title:string
	content:string
	subtitle?:string //可有可无的成员 相当于类型为 string | undefined
}
  • 接口的只读成员：
interface Post {
	title:string
	content:string
	readonly summery:string //赋值之后不可修改
}

const hello:Post = {
	title:'aaa',
	content:'bbb',
	summery:'ccc'
}

hello.summery = 'other'  //err
  • 动态成员

适用于动态成员的对象

interface Cache {
	[prop:string]:string
}

const cache: Cache = {}

cache.foo = 'foo'
cache.bar = 'bar'

# 类

  • 声明一个类:
class Person{
	//与js不同的是，需要在构造函数赋值前先定义一次变量，以规定其变量类型
	name:string
	age:number
	
	constructor(name:string, age:number){
		this.name = name,
		this.age = age
	}
	
	sayHi(msg:string): void {
		console.log(`${this.name},${msg}`)
	}
}
  • 类的访问修饰符
class Person{
	public name:string   //声明公有成员，默认值
	private age:number  //声明私有成员
	protected gender:age //受保护的成员，只允许在子类中访问
	
	constructor(name:string, age:number){
		this.name = name,
		this.age = age
	}
	
	sayHi(msg:string): void {
		console.log(`${this.name},${msg}`)
	}
}
